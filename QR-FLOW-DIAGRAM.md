```
┌─────────────────────────────────────────────────────────────────────────┐

                                    ↓

                                    │
                                    │ Validate
                                    ↓

│                    verifyBookingQR()                                    │

                                    │ Return error
                                    ↓

                                    │ Hiển thị lỗi
                                    ↓
```

## 🔐 Security Flow

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         SECURITY CHECKS                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. QR Format Validation                                               │
│     ✅ Must start with "SMARTPARK_BOOKING" or "SMARTPARK_PASS"         │
│     ✅ Must have correct number of fields (6 for booking, 5 for pass)  │
│     ✅ ID must be numeric                                               │
│                                                                         │
│  2. Database Existence Check                                           │
│     ✅ Record must exist in database                                    │
│     ✅ ID must match                                                    │
│                                                                         │
│  3. Code Verification                                                  │
│     ✅ payment_code in DB must match code in QR                        │
│     ✅ Prevents fake QR codes                                           │
│                                                                         │
│  4. Status Validation                                                  │
│     ✅ Booking: status = "PAID" or "CONFIRMED"                         │
│     ✅ Pass: status = "ACTIVE"                                          │
│                                                                         │
│  5. Date Validation                                                    │
│     ✅ Booking: booking_date >= today                                   │
│     ✅ Pass: end_date >= today                                          │
│                                                                         │
│  6. Optional: One-time Use                                             │
│     ⚠️  Can add "used" flag to prevent reuse                           │
│     ⚠️  Set used=true after first check-in                             │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```
